# TB-10 · Неполные данные (пустой трекер)

**task4:** доп. (деградация) · **сложность:** low · **формат:** v1

Трекер вернул пустой список; в письме — открытый вопрос.

## Проверяемые функции

- деградация без падения
- пометка «данные неполны»
- вопросы из писем

## Проверяемые критерии

- нет падения
- пометка неполноты
- вопрос виден

## Что проверяется (checks)

- пометка «данные неполны»
- деградация без падения
- вопрос из письма виден

## Структура

- `input/` — обезличенные синтетические выгрузки (calendar/tracker/mail/transcript/memory_seed)
- `expected/` — эталонная разметка, зафиксирована ДО оценки фактического вывода
- `meta.json` — параметры прогона (движок, формат, флаги источников)
- `metadata.yaml` — расширенные метаданные сценария (task4 §7)

## Ожидаемое поведение

падение или не пропускать вопроса.

## Возможный отказ

падение или пропуск вопроса

## Источники

Обязательные: calendar, mail, transcript. Опциональные: tracker.

Данные синтетические и обезличенные: проект «Альфа», релиз ALPHA-2026.07, роли вместо ФИО.
