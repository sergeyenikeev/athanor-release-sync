# TB-02 · Критичный блокер релиза

**task4:** СЦ-02 · **сложность:** low · **формат:** v1

Jira-задача в работе; письмо сообщает о проблеме со смежным сервисом; релизное окно близко.

## Проверяемые функции

- сбор контекста
- сводка (блокеры)
- оценка серьёзности
- эскалация

## Проверяемые критерии

- блокер
- серьёзность
- связь с релизом
- эскалация

## Что проверяется (checks)

- блокер из письма попадает в сводку с пометкой
- блокер структурирован (Blocker)

## Структура

- `input/` — обезличенные синтетические выгрузки (calendar/tracker/mail/transcript/memory_seed)
- `expected/` — эталонная разметка, зафиксирована ДО оценки фактического вывода
- `meta.json` — параметры прогона (движок, формат, флаги источников)
- `metadata.yaml` — расширенные метаданные сценария (task4 §7)

## Ожидаемое поведение

не пропускать блокера или неверная серьёзность.

## Возможный отказ

пропуск блокера или неверная серьёзность

## Источники

Обязательные: calendar, tracker, git, mail, transcript. Опциональные: memory.

Данные синтетические и обезличенные: проект «Альфа», релиз ALPHA-2026.07, роли вместо ФИО.
