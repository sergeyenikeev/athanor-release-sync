# Безопасность

## Safety Layer (src/athanor/security.py)
- **Разделение данных и инструкций**: текст писем и расшифровок — ДАННЫЕ.
  Инструкции из них не исполняются; для LLM недоверенный текст оборачивается
  маркерами `<<<BEGIN/END UNTRUSTED DATA>>>` (`wrap_untrusted`).
- **Обнаружение prompt injection**: `scan_untrusted` — 7 паттернов (игнорируй
  инструкции, перешли наружу, покажи промпт/ключ, «ты больше не агент», …).
  Подозрительные фрагменты помечаются в выводе (карантин), не исполняются.
- **Allowlist инструментов**: `ALLOWED_READ_TOOLS` + `ALLOWED_HITL_TOOLS`;
  `is_write_tool` — запись только через HITL. Автоматическое расширение прав запрещено.
- **Маскирование сенсоров**: `mask_secrets` — API-ключи, токены, email, номера карт.
- **Валидация файлов памяти**: `validate_memory_file` — проверка канонического
  заголовка и отсутствие инъекций перед изменением памяти.

## Human-in-the-loop (src/athanor/hitl.py)
Гейт обязателен для: создания задачи, отправки письма, изменения срока, назначения
ответственного, закрытия обязательства, изменения памяти навыка, применения новой
версии навыка. Статусы: `proposed → awaiting_approval → approved → executed`
(или `→ rejected`, `→ failed`). Без подтверждения исполнение → `failed`.

## Аудит
- Память релиза — markdown + `memory/journal.log` (кто/что/когда/почему).
- Реестр версий навыка — `skills/release_sync/versions/registry.json` (history).
- Стоимость LLM — `results/runs/<id>/llm_cost.log`.
- Каждый прогон сохраняет вход, выход, память, черновики, метрики.
