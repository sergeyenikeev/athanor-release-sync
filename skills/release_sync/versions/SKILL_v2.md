# Навык release_sync (v2)

Изменения против v1 (по обратной связи пользователя, проверено на 3 контрольных
кейсах TB-01/TB-02/TB-04 перед promote_to_stable; откат: rollback до v1):

1. **Порядок секций сводки**: конфликты и блокеры — первыми, затем обязательства;
   статусы задач — не более трёх строк (остальное по запросу); секция
   «открытые вопросы» показывается только при наличии.
2. **Компактность**: целевой объём сводки ≤ 12 строк.
3. Всё остальное — без изменений (процесс, правила безопасности, HITL,
   форматы поручений и памяти идентичны v1, см. versions/SKILL_v1.md).

## Процесс
Идентичен v1 (шаги 1–7), за исключением шага 3: порядок секций —
конфликты → блокеры → обязательства → статусы (≤3) → полнота данных.
